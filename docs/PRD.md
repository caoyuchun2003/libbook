# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
宇春书城 - 图书管理系统

### 1.2 产品定位
宇春书城 是一个现代化的图书管理系统，旨在帮助图书馆、学校或组织高效管理图书资源。

### 1.3 目标用户
- **主要用户**：图书管理员、学生、教师
- **次要用户**：系统管理员

## 2. 用户角色

### 2.1 普通用户
- 浏览图书目录
- 搜索图书
- 查看图书详情

### 2.2 管理员
- 所有普通用户功能
- 添加/编辑/删除图书
- 管理图书分类
- 管理用户

## 3. 功能需求

### 3.1 用户认证模块

#### 3.1.1 用户注册
- **优先级**：高
- **描述**：新用户可以通过邮箱和密码注册账号
- **输入**：姓名、邮箱、密码、确认密码
- **验证**：
  - 邮箱格式验证
  - 密码长度至少8位
  - 密码确认匹配
  - 邮箱唯一性检查

#### 3.1.2 用户登录
- **优先级**：高
- **描述**：已注册用户通过邮箱和密码登录
- **输入**：邮箱、密码
- **输出**：认证令牌（Token）

#### 3.1.3 用户登出
- **优先级**：中
- **描述**：用户主动退出登录，清除认证令牌

### 3.2 图书管理模块

#### 3.2.1 图书列表
- **优先级**：高
- **描述**：展示所有图书，支持分页、搜索、筛选
- **功能**：
  - 分页显示（默认每页15条）
  - 按标题、作者、ISBN搜索
  - 按分类筛选
  - 显示图书基本信息（标题、作者、ISBN、分类、可借数量）

#### 3.2.2 图书详情
- **优先级**：高
- **描述**：查看单本图书的详细信息
- **信息包含**：
  - 基本信息：标题、作者、ISBN、分类、描述、封面、售价、总数量、可借数量、出版日期
  - 内容简介：富文本格式的内容介绍
  - 作者简介：富文本格式的作者介绍
  - 目录：多级章节目录（可点击跳转）
  - 章节内容：完整的章节正文内容（富文本）

#### 3.2.3 添加图书（管理员）
- **优先级**：高
- **描述**：管理员可以添加新图书
- **必填字段**：标题、作者、ISBN、分类、总数量
- **可选字段**：描述、封面URL、售价、内容简介、作者简介、出版日期

#### 3.2.4 编辑图书（管理员）
- **优先级**：中
- **描述**：管理员可以修改图书信息

#### 3.2.5 删除图书（管理员）
- **优先级**：中
- **描述**：管理员可以删除图书

### 3.3 分类管理模块

#### 3.3.1 分类列表
- **优先级**：中
- **描述**：显示所有图书分类

#### 3.3.2 添加分类（管理员）
- **优先级**：中
- **描述**：管理员可以添加新的图书分类

#### 3.3.3 编辑/删除分类（管理员）
- **优先级**：低
- **描述**：管理员可以修改或删除分类

### 3.4 章节管理模块（管理员）

#### 3.4.1 章节列表
- **优先级**：高
- **描述**：管理员可以查看图书的所有章节，以树形结构展示
- **功能**：
  - 显示章节层级关系
  - 支持展开/折叠子章节
  - 显示章节标题和顺序

#### 3.4.2 添加章节
- **优先级**：高
- **描述**：管理员可以为图书添加章节
- **功能**：
  - 添加一级章节
  - 为章节添加子章节（支持多级）
  - 设置章节标题和正文内容（富文本）
  - 自动设置章节顺序

#### 3.4.3 编辑章节
- **优先级**：中
- **描述**：管理员可以修改章节标题和正文内容
- **功能**：
  - 修改章节标题
  - 修改章节正文内容（富文本）
  - 调整章节的父章节关系

#### 3.4.4 删除章节
- **优先级**：中
- **描述**：管理员可以删除章节
- **注意**：删除章节会级联删除其所有子章节

#### 3.4.5 章节排序
- **优先级**：低
- **描述**：管理员可以调整章节的显示顺序

### 3.5 首页/仪表板

#### 3.5.1 数据统计
- **优先级**：中
- **描述**：显示关键数据指标
- **指标**：
  - 图书总数
  - 可借图书数量
  - 图书分类数量

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间 < 2秒
- API 响应时间 < 500ms
- 支持至少1000本图书的搜索

### 4.2 安全需求
- 用户密码加密存储
- API 使用 Token 认证
- 防止 SQL 注入和 XSS 攻击
- CORS 配置

### 4.3 可用性需求
- 响应式设计，支持移动端访问
- 友好的错误提示
- 操作确认提示（删除、归还等）

### 4.4 兼容性需求
- 支持现代浏览器（Chrome、Firefox、Safari、Edge）
- 后端支持 PHP 8.1+
- 前端支持 ES6+

## 5. 优先级规划

### 第一阶段（MVP）
1. 用户注册/登录
2. 图书列表和搜索
3. 图书详情页
4. 图书分类管理

### 第二阶段
1. 管理员功能
2. 数据统计
3. 用户管理

### 第三阶段
1. 高级搜索和筛选
2. 批量操作
3. 数据导出
4. 图书评价功能

## 6. 技术约束

- 后端：Laravel 10+
- 前端：Vue.js 3 + Vite
- 数据库：MySQL/PostgreSQL
- 认证：Laravel Sanctum
- UI 框架：Element Plus
